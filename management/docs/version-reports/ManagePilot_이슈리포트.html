전체 파이프라인

아래 각 단계에서 보안·권한·인코딩 이슈가 순차적으로 발생했다.

보안 xlsx (IRM)→ 사용자 PC Excel(COM)→ 평문 CSV→ 공유폴더/로컬→ 서버 파싱 · 대시보드
1

IRM 암호화로 서버가 엑셀을 못 읽음

해결
증상
회사 정보보호(IRM/AIP) 라벨이 걸린 Daily Report를 서버 파서(SheetJS)가 열지 못함 — 시트가 Sheet1 하나로만 인식.
원인
서버 프로세스는 복호화 권한이 없음. IRM 파일은 권한 있는 사용자 계정의 Office만 복호화 가능.
대응
권한 계정 PC에서 Excel(COM)로 열어 평문 CSV로 추출 → 서버는 CSV만 읽도록 구조 변경. CSV는 구조상 보안 라벨을 가질 수 없어 항상 평문.
2

“복사 금지” 권한으로 시트 내보내기 차단

해결
증상
Excel COM으로 시트 복사 시 이 시트를 복사할 수 없습니다.
원인
IRM 권한이 “보기 허용, 복사/추출 금지”로 설정됨 → Worksheet.Copy() 자체가 차단.
대응
시트를 복사하지 않고 셀 값(UsedRange.Value2)을 직접 읽어 CSV로 기록. 값 조회는 허용되어 우회 성공.
3

PowerShell 스크립트의 한글이 깨짐

해결
증상
스크립트 실행 시 구문 오류, 또는 경로가 ?꿃 궲 ?대뚉 처럼 깨짐.
A positional parameter cannot be found... / date sheet not found
원인
Windows PowerShell 5.1은 BOM 없는 .ps1을 ANSI(CP949)로 읽음 → UTF-8 한글이 오독.
대응
① 시트 인식 패턴을 유니코드 코드포인트([char]0xC6D4)로 구성해 스크립트를 순수 ASCII화.
② 한글 경로는 별도 manage-config.txt(UTF-8)로 분리, ReadAllLines(…, UTF8)로 강제 디코딩.
4

공유폴더 목록 조회 권한 없음

해결
증상
폴더에서 최신 파일을 찾을 때 Get-ChildItem : Access is denied.
원인
해당 네트워크 공유는 파일 열기는 허용, 폴더 목록 조회는 거부.
대응
폴더를 뒤지지 않고 현재 월로 파일명을 직접 생성({M}·{YY})해 바로 열기. 없으면 전월/전전월 자동 시도.
5

배치(.bat)에서 한글 경로 처리 실패

해결
증상
'어' is not recognized… / 공백 경로가 조각남(생산팀\1공장).
원인
cmd의 한글·공백 경로 인용 처리 취약.
대응
bat에서 한글을 완전히 제거. bat은 ASCII 런처로 manage-export.ps1만 호출하고, 경로는 config.txt에 위치.
6

월이 바뀌면 파일명이 달라져 못 읽음

해결
증상
7월이 되자 …_6월.xlsx 고정 경로로는 파일을 못 찾음.
대응
경로에 {M}/{YY} 자리표시자 도입 → 실행 시점의 월/연도로 자동 완성. 시트 선택도 최신 날짜 시트 자동 인식.
7

파일 잠김 · 열기 암호 (외부 요인)

외부 의존
증상
액세스할 수 없습니다 — 읽기 전용이거나 암호화되어 있습니다.
원인
① 해당 파일을 다른 사용자가 편집 중(잠김) ② 열기 암호가 걸린 파일 ③ 원본 저장 방식 변경.
대응
스크립트: 최신월이 안 열리면 전월 파일 자동 시도 + 어떤 파일을 열었는지 [SRC] 로그.
근본 해결은 운영측: 파일 닫기 / 열기 암호 제거 / IT에 보안 라벨 제외 요청.

종합 · 남은 리스크

현재 상태
코드·인코딩·권한 이슈(1~6)는 모두 해결. 정상 파일이면 추출→서버 반영까지 자동 동작.
잔여 리스크
운영 의존: ① 추출 PC가 매일 로그온 상태여야 IRM 복호화 가능 ② 원본 파일이 잠김/암호면 열 수 없음.
권고
전용 PC 상시 로그온 + 작업 스케줄러(로그온 시 실행) 등록. 장기적으로는 IT에 해당 보고서 IRM 제외 요청이 가장 안정적.
