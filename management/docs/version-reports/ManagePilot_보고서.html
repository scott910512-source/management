모듈 분리 · 보안 데이터 연동 기반

TotalPilot 허브에서 ManagePilot(생산)·StockPilot(재고) 모듈을 분리하고, 회사 보안(IRM)으로 잠긴 Daily Report 엑셀을 자동으로 평문화하여 서버가 읽을 수 있게 만드는 데이터 파이프라인을 구축했다.

핵심 컴포넌트

1단계 주요 기술 컴포넌트
허브/모듈로그인 → /hub(TotalPilot)에서 모듈 선택. 사이드바 로고 클릭으로 ManagePilot ↔ StockPilot 즉시 전환.
멀티 공장데이터는 server/data/1공장·2공장 폴더로 분리. 총괄관리자(plantScope=all)는 대시보드 탭으로 공장 전환.
로그인 이력전역 login_logs CSV에 성공/실패/차단 기록, 관리자 조회.
IRM 우회보안 라벨 엑셀은 SheetJS로 직접 못 읽으므로, 권한 계정 PC의 Excel(COM)로 평문 CSV를 추출.

IRM 보안 엑셀 → 평문 CSV 파이프라인

회사 정보보호(IRM/AIP) 라벨이 적용된 Daily Report는 서버 프로세스가 복호화할 수 없다. "복사 금지" 권한 때문에 시트 복사조차 막혀, 최종적으로 셀 값을 직접 읽어 UTF-8 CSV로 내보내는 PowerShell 스크립트로 해결했다. CSV는 구조상 보안 라벨을 가질 수 없어 항상 평문이다.

보안 xlsx→ Excel COM (권한계정)→ UsedRange 값 읽기→ daily-latest.csv
batch-yield.csv→ 서버 파싱 
# export-daily-report-csv.ps1 — 스마트 자동 선택 (ASCII 전용 소스)
# 최신 날짜시트 → daily-latest.csv / 배치별수율 시트 → batch-yield.csv
foreach ($ws in $wb.Worksheets) {
  $n = ($ws.Name -replace $FWSP_RE, ' ').Trim()
  if ($n -match $DATE_RE) {           # "N월 N일" 중 최신
    $key = [int]$matches[1]*100 + [int]$matches[2]
    if ($key -gt $latestKey) { $latestKey=$key; $latestWs=$ws }
  } elseif ($n -match $BATCH_RE) { $batchWs = $ws }
}
한글 정규식이 ANSI(CP949) 인코딩에서 깨지는 문제 → 월/일·배치별·수율 패턴을 [char]0xC6D4 같은 유니코드 코드포인트로 구성해 소스를 ASCII 전용으로 만들어 인코딩 독립성을 확보했다. 작업 스케줄러로 매일 자동 추출.

공장별 셀 매핑 (관리자 설정)

공장마다 양식의 열 위치가 다르므로, 코드 수정 없이 엑셀 열 문자(B·K·AF…)를 관리자가 지정하도록 구성했다. 행은 제품명으로 자동 탐색하고, 특수 양식(BTBAS 합계 재고 등)은 AN10 같은 절대 셀 오버라이드를 지원한다.

2

종합현황 대시보드 · 데이터 파싱

평문 CSV를 셀 좌표 기반으로 파싱하여, 공장별 생산현황·재고·달성율을 한 화면에 집계하는 종합현황 대시보드를 구성했다.

일일 종합현황 파싱 (daily-latest.csv)

제품 행 앵커링 + 열 매핑
행 탐색제품명(예: CpHf)을 productCol에서 찾아 해당 행을 앵커 → 행이 밀려도 안전.
생산 KPI오늘 생산량·6월 계획/실적/달성율·연간 계획/실적/달성율·수율(월/년)을 지정 열에서 추출.
재고충전완료·출하·잔여수량(can) + 절대셀 오버라이드. 안전재고(개월) 대비 잔여 개월수 계산.
// production.js — parseDailyCsv (셀 좌표 파싱)
const pr = dFindRow(grid, idx.product, p);   // 제품 행
byProduct[p] = {
  monthActual: dNum(g(pr, idx.mAct)),
  monthRate:   dPct(g(pr, idx.mRate)),   // 0.88 → 88.0%
  yield:       dPct(g(pr+yieldOffset, idx.mAct)),
  inventory: { total: invVal(ov.total, idx.invTotal) },
};

잔여 개월수 (재고 기준정보)

관리자가 품목별 연간계획(kg)·포장단위(kg)를 입력하면 월 소비량(can)을 자동 계산하고, 잔여수량으로 소진 개월수를 산출한다. 안전재고(개월) 미만이면 요약카드에서 "재고관리 필요"로 경고한다.

// 월 소비량(can) = 연간계획(kg) ÷ 12 ÷ 포장단위(kg)
const monthlyUse = annualPlan / 12 / packageUnit;
inv.remainingMonths = inv.total / monthlyUse;       // 잔여 개월수
inv.belowSafety = inv.remainingMonths < (safetyMonths ?? 2);

월별 추이 · 시각화 고도화

배치별 수율 시트를 파싱해 월별 생산량·수율·배치수를 집계하고, 품목별 선그래프와 클릭형 내역표로 추세를 시각화했다.

배치시트 파싱 (batch-yield.csv)

월 블록(N월 ~ Sub total)을 인식해, 소계 행에서 월 생산량·수율을, Batch no 열의 # 포함 행으로 배치 수를 집계한다. 소계가 월 열 또는 품목별 Batch no 열 어디에 있어도 인식하도록 처리해 공장별 양식 차이를 흡수했다.

// mergeBatchYield — 월/품목 소계 인식
const rowIsSub = isSub(monthCell) ||
  names.some(p => isSub(cell(r, prods[p].no)));
if (rowIsSub && curMonth) {
  out[p][curMonth] = { actual, yield, batchCount: counts[p] };
}

월별 선그래프

MonthlyLineChart (SVG)
품목별 선생산량 그래프는 품목별 선 + 총 생산량 굵은 선. 수율 그래프는 품목별 %.
Y축 스케일최대치를 총합 × 1.1로 잡아 총생산량 선이 잘리지 않게 구성.
결측 처리10% 미만·음수·오류값(-2.15E+09)은 0과 동일하게 공백 처리, 선을 끊어 추세 왜곡 방지.
미래월현재월 이후는 회색 "데이터 없음" 영역으로 표시.
// 10% 미만/음수/오류값은 공백(null)
const val = (p, m) => {
  const v = byProduct[p]?.monthlyData?.[m-1]?.actual;
  return (v == null || v < 10) ? null : v;
};
// Y 최대 = 월별 총합의 최대 × 1.1
maxV = Math.max(1, ...totals.map(t => t||0)) * 1.1;
