1. 프로젝트 개요

Windows VDI(가상 데스크톱) 환경에서 자주 쓰는 웹사이트, 프로그램, 폴더를 키보드 단축키 한 번 또는 작은 런처 창으로 즉시 실행하는 도구입니다.

핵심 설계 원칙

  • 외부 라이브러리 0개 — 파이썬 표준 라이브러리(ctypes, tkinter, webbrowser, subprocess)만 사용
  • 오프라인 대응 — wheel 다운로드/설치 불필요. 폐쇄망 VDI에서 파이썬만 있으면 동작
  • 관리자 권한 불필요 — Win32 RegisterHotKey는 일반 권한으로 동작
  • 가벼움 — RAM 약 30MB, idle 시 CPU 0%

2. 동작 원리

① 전역 핫키 (Global Hotkey)

ctypes로 Win32 API RegisterHotKey를 직접 호출합니다. 백그라운드 스레드에서 메시지 루프(GetMessageW)를 돌며 WM_HOTKEY 메시지를 받아 등록된 콜백을 실행합니다.

② 런처 창

tkinter로 만든 작은 창. Ctrl+Alt+Space로 띄우고 숫자키·클릭으로 실행합니다. 핫키 스레드와 UI 스레드 간 통신은 queue.Queue로 안전하게 처리합니다.

③ 실행 메커니즘

종류사용 함수특징
urlwebbrowser.open기본 브라우저로 열기
program / file / folderShellExecuteWApp Paths·PATH·파일연결 해석, 이름만으로도 실행
commandsubprocess.Popen셸 명령 실행
핵심 포인트: 런처는 "관리자" 역할만 하고, 실제 프로그램은 각각 독립 프로세스로 실행됩니다. 따라서 런처와 다른 프로그램이 서로 영향 없이 동시에 돌아갑니다.

4. 사용 방법

  1. 1파일 배치 — vdi-launcher 폴더를 VDI에 복사
  2. 2설정 수정 — config.json에 항목 작성
  3. 3실행 — start_launcher.bat 더블클릭
  4. 4자동 시작(선택) — Win+R → shell:startup 폴더에 .bat 바로가기 복사

종료 방법

  • 런처 창의 빨간 종료 버튼
  • 전역 핫키 Ctrl+Alt+Q (quit_hotkey 설정 시)
  • 명령어 taskkill /f /im pythonw.exe

5. 설정(config.json) 작성법

{
  "toggle_hotkey": "ctrl+alt+space",
  "quit_hotkey": "ctrl+alt+q",
  "items": [
    { "name": "사내 포털", "type": "url",     "target": "https://portal.example.com", "hotkey": "ctrl+alt+1" },
    { "name": "Chrome",   "type": "program", "target": "chrome",                     "hotkey": "ctrl+alt+2" },
    { "name": "CUBE",     "type": "program", "target": "C:\\CUBE\\CUBE.exe",         "hotkey": "ctrl+alt+3", "admin": true },
    { "name": "공유폴더", "type": "folder",  "target": "C:\\CUBE",                   "hotkey": "ctrl+alt+4" }
  ]
}
경로 작성 3대 규칙
  • target은 대괄호 없는 문자열 — "C:\\..." (O) / ["C:\\..."] (X)
  • 역슬래시는 두 개 — C:\\CUBE\\CUBE.exe (O) / C://CUBE (X)
  • 폴더 위치는 target에 — args는 프로그램 실행 옵션 전용(폴더 아님)

type 종류

type의미target 예시
url웹사이트https://...
program프로그램chrome, C:\\CUBE\\CUBE.exe
folder폴더 열기C:\\CUBE
command셸 명령cmd /c ipconfig

6. 핫키 표기법

+로 조합. 수정자(ctrl, alt, shift, win) + 일반 키.

  • 일반 키: a~z, 0~9, f1~f12
  • 특수 키: space, enter, esc, tab, home, end, 방향키 등
  • 예: ctrl+alt+1, ctrl+shift+p, win+f2
💡 핫키 하나에 수정자 1개 이상 권장. 단독 키는 Windows가 잘 잡지 못합니다.

7. 레슨 런 (Lessons Learned)

이 프로젝트를 만들며 겪은 문제와 해결 과정 — 같은 실수를 반복하지 않기 위한 기록.

① 오프라인 VDI = 외부 라이브러리 부담
문제 keyboard·pynput 같은 라이브러리는 wheel 오프라인 다운로드 필요 + 인증/안정성 부담
해결 ctypes로 Win32 API 직접 호출 → 표준 라이브러리만으로 구현, 설치 불필요
② 프로그램 이름만으로 실행이 안 됨
문제 subprocess.Popen은 PATH 등록/전체 경로만 동작 → chrome 등 실패
해결 ShellExecute로 변경 → App Paths 레지스트리·PATH·파일연결까지 해석
③ 슬래시 방향·따옴표 실수
문제 C:/폴더/app.exe처럼 /로 적으면 Windows가 못 찾음
해결 경로 자동 보정(/→\, 따옴표 제거) 로직 추가
④ 특정 exe가 실행되다 꺼짐
문제 작업 폴더 미지정 → 상대경로 DLL/리소스 못 찾음
해결 exe의 폴더를 working directory로 지정 (더블클릭과 동일 환경)
⑤ target에 대괄호·args에 폴더 넣는 혼동
문제 "target": ["C://CUBE//CUBE.exe"] — 대괄호=배열, 슬래시 오류
해결 target은 단일 문자열, args는 실행 옵션 전용임을 명확히 (문서·예시 보강)
⑥ 종료 수단 부재
문제 창 닫기는 숨기기만 → 작업관리자로만 종료 가능
해결 종료 버튼 + 전역 종료 핫키(Ctrl+Alt+Q) 추가

핵심 교훈 요약

  • 제약(오프라인)을 먼저 파악하면 설계가 단순해진다 → "표준 라이브러리만" 결정
  • Windows 실행은 ShellExecute가 정답 — 탐색기 더블클릭과 동일한 동작
  • 사용자 입력은 너그럽게 보정 — 슬래시·따옴표·공백 자동 처리
  • 오류 메시지를 명확하게 — 오류 코드를 한글로 풀어 원인 안내

8. 자주 묻는 질문

RAM을 많이 잡아먹나요?

아니요. 약 30MB로 매우 가볍습니다 (Chrome은 200~500MB). idle 시 CPU는 0%입니다.

Python이 켜져있으면 다른 프로그램 못 돌리나요?

전혀 상관없습니다. Python 프로세스는 런처 자체만 실행하고, 실행되는 프로그램은 각각 독립 프로세스로 시작됩니다. 동시 실행에 문제 없습니다.

관리자 권한이 필요한가요?

아니요. 다만 관리자 권한이 필요한 프로그램은 config에 "admin": true를 추가하세요.

핫키가 안 먹어요

다른 프로그램이 같은 조합을 쓰는 경우입니다. config에서 다른 조합으로 바꾸고 "새로고침"하세요.

실무 적용 시나리오

  • 업무 시작 자동화 — 한 핫키로 매일 쓰는 사내 시스템 일괄 실행
  • 부서 표준 배포 — 팀 공용 config.json 배포로 단축키 환경 통일
  • 교육/온보딩 — 신규 입사자에게 사내 시스템 바로가기 모음 제공
  • 반복 작업 단축 — 자주 여는 공유 폴더·보고서 양식을 핫키화

기술 확장 방향

  • 크로스 플랫폼 — Linux VDI 지원(X11 핫키) 추가로 환경 확대
  • 설정 GUI — config.json 직접 편집 대신 폼 기반 설정 화면
  • 매크로 연동 — 단순 실행을 넘어 키 입력 자동화까지 확장
  • 프로파일 전환 — 프로젝트별 단축키 세트 전환 기능
가장 추천하는 다음 스텝: "찾아보기 버튼" + "숫자패드 지원" — 둘 다 난이도가 낮으면서 경로 입력 실수를 없애고 사용 편의를 크게 높입니다.
