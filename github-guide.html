깃허브 기본구성

혼자 쓰는 사람 기준 — 딱 이것만 알면 됩니다.

레포지토리
→
브랜치
→
작업 / 커밋
→
머지

01

레포지토리 (Repository)

프로젝트 하나 = 레포지토리 하나.
모든 파일과 변경 이력이 여기에 저장됩니다.

예시 — "쇼핑몰 앱"을 만든다면 my-shop 이라는 레포지토리 하나를 만들면 됩니다.

02

브랜치 (Branch)

현재 코드를 복사해서 만드는 독립 작업공간.
기능 하나를 건드릴 때 main을 직접 수정하지 않고, 브랜치를 따서 작업합니다.

# main 기준으로 새 브랜치 만들기 git checkout -b feature/로그인

main = 완성된 안정 코드만 두는 곳
feature/xxx = 기능 작업할 때 쓰는 임시 공간

03

작업 / 커밋 (Commit)

브랜치에서 파일을 수정하고 커밋으로 저장합니다.
커밋은 "이 시점 코드를 저장해 둔 것" — 나중에 되돌릴 수도 있습니다.

# 변경된 파일 전체 올리기 git add . # 커밋 (뭘 했는지 한 줄로) git commit -m "로그인 화면 추가" # GitHub에 올리기 git push origin feature/로그인

04

머지 (Merge)

작업이 끝난 브랜치를 main에 합치는 것.
혼자 쓸 때는 GitHub에서 버튼 하나로 끝납니다.

# GitHub에서 Pull Request 생성 후 # "Merge pull request" 버튼 클릭 # 로컬도 최신화하고 싶으면 git checkout main git pull origin main

혼자 쓰면 리뷰 없이 바로 머지해도 됩니다.
머지 후 브랜치는 삭제해서 깔끔하게 유지하세요.
