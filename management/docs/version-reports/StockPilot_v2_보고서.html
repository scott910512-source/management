버전 컨셉 · 기술 배경

입고일별 Lot 단위로 재고를 분리 추적하고, 품목 기준정보(단위·안전재고·업체·제품)를 마스터 테이블로 체계화

Lot 기반 데이터 구조

// raw_materials 테이블 컬럼 (CSV 행 형태)
{ id: 'rm-xxxx', itemId: 'it-xxxx', lotNo: 'A2401',
  receivedDate: '2024-01-15', quantity: '200', weight: '200',
  vendor: 'A업체', product: '제품A',
  createdBy: 'user1', createdAt: '2024-01-15T09:00:00Z' }

// items (품목 기준정보) 마스터 테이블
{ id: 'it-xxxx', category: 'raw', name: '헥산',
  unit: 'L', safetyStock: '100', warningPct: '150',
  vendor: 'A업체', product: '제품A' }

이중 안전재고 계산

// 총 잔량 집계 후 두 가지 기준으로 판단
const totalQty = lots.reduce((s, l) => s + num(l.quantity), 0);
const safetyStock = num(item.safetyStock);      // 절대 수량 기준
const warningPct = num(item.warningPct) / 100;  // % 기준 (예: 150%)
const isLow = totalQty < safetyStock || totalQty < safetyStock * warningPct;
// warningPct=150이면 안전재고의 1.5배 미만 시 경고
테이블 구조
items품목 기준정보 마스터 — 단위·안전재고·업체·제품 저장. 트랜잭션이 참조하는 기준점.
raw_materials원재료 Lot별 재고 — itemId 참조, receivedDate·quantity·weight 관리.
sub_materials부재료 Lot별 재고 — 구조 동일. raw/sub를 라우터·저장소 모두 분리.
canistersCanister 용기 — content(내용물), weight, size별 최대무게 비교로 90% 경고.
transactions수불 이력 — type(입고/출고), materialId, quantity, batchNo 통합 기록.
2

주요 기능 내용

실제 구현된 기능과 기술적 처리 방식

품목 기준정보
items 마스터 테이블에 단위·안전재고·업체·제품 사전 등록. 입고/출고 시 품목 선택만으로 양식 자동 채움.
Lot 단위 입고
입고 시 receivedDate + lotNo를 키로 Lot 생성. 출고 시 해당 Lot의 quantity를 직접 차감하여 실시간 추적.
이중 안전재고
safetyStock(절대 기준수량)과 warningPct(% 기준) 두 가지로 부족 감지. % 기준은 "안전재고의 N% 미만" 조건.
Canister 내용물·무게
canisters 테이블에 content, weight 저장. 사이즈(200L/300L 등)별 최대무게 설정 → 90% 이상 시 경고.
용기이력카드
용기별 반입·반출 transactions를 필터링해 이력 카드로 조회. 이동·세척·파기까지 추적.
수불이력 통합 조회
원재료·부재료·Canister 수불을 transactions 한 테이블에서 materialType으로 구분 조회.

기능 강화 · 인터락 (안전장치)

실수·사고·무단 접근을 막는 서버 레벨 안전장치

잔량 직접 차감
출고 요청 시 해당 Lot의 quantity -= 출고량 즉시 반영. 음수 방지는 Math.max(0, ...)로 처리.
이중 안전재고
safetyStock 절대값 + warningPct % 기준 두 가지 중 하나라도 미달 시 경고 발생.
입고일 필수 검증
입고일 없으면 서버에서 badRequest('입고일을 입력하세요') 반환. FIFO 계산의 기준값이 되므로 필수.
5

버그 수정 · 원인 분석

발생 원인과 해결 방법

단위 선택 미제공
초기엔 단위가 고정(kg)이었음 → items 마스터에 unit 필드 추가, kg·ea·L·기타 선택 가능하도록 변경.
동시 쓰기 충돌
withLock 뮤텍스를 raw_materials·sub_materials 별도 적용. 테이블별 독립 잠금 구조.
빈 Lot 목록 노출
잔량 0인 Lot도 목록에 표시되어 혼란 → quantity <= 0 필터로 기본 숨김, 이력 조회에선 포함.

학습 · 기술 포인트

이 버전에서 익힌 핵심 기술 개념

기준정보(마스터)와 수불(트랜잭션)을 별도 테이블로 분리하는 패턴. 마스터 변경이 이력에 영향 없도록 설계.

테이블 단위 뮤텍스로 서로 다른 테이블 동시 접근은 허용, 같은 테이블 동시 접근만 직렬화.
