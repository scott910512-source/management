그림 1. 보안 엑셀 → 평문 CSV → 서버 파싱 → 대시보드 파이프라인
1

구축 중 겪은 문제점

보안 엑셀을 서버가 자동으로 읽게 만드는 과정에서 발생한 이슈와 해결.

보안(IRM) 암호화로 서버가 엑셀을 못 읽음 해결
원인
라벨이 걸린 파일은 서버가 복호화 권한이 없음.
해결
권한 계정 PC의 Excel로 열어 평문 CSV로 추출. CSV는 라벨을 못 가져 항상 평문.
“복사 금지” 권한으로 시트 내보내기 차단 해결
해결
시트 복사 대신 셀 값만 직접 읽어 CSV로 기록.
# 값 조회는 허용됨 → UsedRange 값을 그대로 CSV로
function Export-SheetCsv($ws,$out){
  $ur=$ws.UsedRange; $vals=$ur.Value2
  for($r=1;$r-le $ur.Rows.Count;$r++){
    for($c=1;$c-le $ur.Columns.Count;$c++){ ... $vals.GetValue($r,$c) ... }
  }
  [IO.File]::WriteAllText($out,$sb,New-Object Text.UTF8Encoding($true))
}
시트 복사(Copy) 대신 셀 값(Value2)만 읽어 “복사 금지” 우회.
한글 인코딩 깨짐 (스크립트·배치) 해결
해결
실행 스크립트는 ASCII로, 한글 경로는 설정 텍스트파일(UTF-8)로 분리해 강제 UTF-8로 읽음.
# 한글 경로는 스크립트 밖(config)에서 강제 UTF-8 로 읽는다
$lines = [IO.File]::ReadAllLines($cfg, [Text.Encoding]::UTF8)
# 시트명 한글 패턴도 코드포인트로 (인코딩 무관)
$WOL=[char]0xC6D4; $IL=[char]0xC77C  # 월, 일
공유폴더 목록 조회 차단 · 월 변경 해결
해결
폴더를 뒤지지 않고 현재 월로 파일명을 만들어 바로 열기. 없으면 전월 자동 시도.
# {연}/{월} 자리표시자 → 실행 시점 값으로 치환 (폴더 목록조회 불필요)
foreach($off in 0,-1,-2){
  $d=(Get-Date).AddMonths($off)
  $p=$pattern -replace '\{M\}',$d.Month -replace '\{YYYY\}',$d.Year
  $names += $p    # 이번달, 전월, 전전월 순으로 시도
}
공장·양식 차이 해결
해결
공장별 셀 매핑(열 위치를 관리자 설정에서 지정) + 최신 날짜시트/배치시트 자동 인식. 제품명으로 행 자동 탐색이라 행이 밀려도 안전.
원본 파일 잠김·열기 암호 외부 요인
완화
전월 파일 자동 시도 + 연 파일 로그. 근본 해결은 파일 닫기/암호 제거(운영 측).
2

운영하면서 놓치면 안 되는 부분

아래 설정이 비어 있으면 해당 지표·경고가 표시되지 않는다. 최초 1회 세팅 필수.

그림 2. 상단 요약 카드 — 수율 주의 품목·재고관리 필요는 아래 설정이 있어야 판정됨
📉 수율 주의 품목 (경고) 설정 전제
동작
품목별 실적 수율 < 목표 수율이면 요약카드·롤링배너·경고이력에 자동 표기(-N.N%p).
전제
원본에 목표 수율 존재 + 셀 매핑에서 수율 열이 맞아야 함.
// 서버: 수율 미달·재고 부족을 자동 도출
if (d.yield < d.yieldTarget) {
  const gap = d.yieldTarget - d.yield;
  push({ level: gap >= 5 ? 'error' : 'warn',   // 5%p↑ 빨강
         msg: `${p} 수율 미달 -${gap}%p` });
}
📦 재고관리 필요 (경고) 기준정보 필수
동작
잔여 개월수 < 안전재고(개월)이면 “재고관리 필요” 경고. 잔여(개월) 색상도 이 기준으로 갈림.
전제
관리자 설정 > 재고 기준정보에 연간계획·포장단위·안전재고 입력. 미입력 시 잔여개월 계산 불가 → 경고 안 뜸.
// 월 소비량(can) = 연간계획(kg) ÷ 12 ÷ 포장단위(kg)
monthlyUse = annualPlan / 12 / packageUnit;
inv.remainingMonths = inv.total / monthlyUse;
inv.belowSafety = inv.remainingMonths < (safetyMonths ?? 2);  // 기본 2개월
🧭 셀 매핑 (양식 연동) 공장별 필수
필요
공장마다 열 위치가 달라 공장별 열 지정 필요. 월별 추이·배치수는 배치시트 매핑까지 넣어야 채워짐.
확인
미리보기로 인식 품목·추이·불일치 확인 후 저장.
🔁 데이터 갱신 흐름 운영 리듬
  • 추출 스크립트가 실행돼야 CSV 최신화 (수동 또는 스케줄러).
  • 화면은 CSV를 읽음 → 스크립트 안 돌면 화면도 그대로.
  • 추출 PC가 로그온·전원 ON 상태여야 보안 복호화가 됨.
  • 새로고침/수동갱신/자동(1시간)은 그 시점 CSV를 다시 읽음.
🗓️ 기준월(월말 스냅샷) · 경고 이력 참고
스냅샷
월말 화면을 연-월별로 아카이브. 원본 시트를 지워도 유지. 상단 기준월에서 선택.
이력
수율 미달·재고 부족을 (공장·기준일·품목)당 하루 1회 기록. 단, 화면을 한 번은 열어야 기록됨.
3

데이터 연동 시 주의사항

원본 양식·환경이 바뀌면 아래를 점검한다.

  • 시트 이름 규칙: 날짜시트(N월 N일)·배치시트(원료 사용량) 이름 유지 → 자동 인식.
  • 열 위치 변경: 열 추가/이동 시 셀 매핑 재조정.
  • 품목명 일치: 종합현황 품목명과 배치시트 품목명 철자 동일해야 월별 추이 연결.
  • 단위 일관성: 수율은 비율(0.78) 기준으로 ×100 처리. 원본이 % 숫자면 조정.
  • 오류·극소값: 음수·오류값·10% 미만은 0으로 간주해 그래프 공백 처리(왜곡 방지).
  • 파일 잠김: 원본 편집 중이면 추출 실패 가능 → 추출 시간대를 편집과 분리.
