버전 컨셉 · 기술 배경

경고·퀵메뉴·Task를 한 화면으로 통합하고, 선입선출(FIFO) 위반을 서버에서 자동 감지하여 409로 차단

FIFO 위반 감지 알고리즘

// server/src/routes/transactions.js — 선입선출 검증
function findEarlierLot(lots, currentLotId, currentReceivedDate) {
  return lots.find(l =>
    l.id !== currentLotId &&
    num(l.quantity) > 0 &&
    l.receivedDate < currentReceivedDate  // 더 오래된 Lot 존재
  );
}

// 출고 처리 시 FIFO 위반 감지
if (!force) {
  const earlier = findEarlierLot(lots, lotId, lot.receivedDate);
  if (earlier) {
    recordAnomaly(req.plant, { /* FIFO 위반 자동 기록 */ });
    throw { status: 409, earlierLot: earlier };  // 프론트에서 경고 표시
  }
}
// force=true 시 → anomalies 테이블에 이상발생 자동 기록

주요 기능 내용

실제 구현된 기능과 기술적 처리 방식

FIFO 인터락
출고 시 findEarlierLot()로 더 오래된 Lot 존재 여부 확인 → 있으면 409 반환. 프론트에서 경고 다이얼로그 표시.
이상발생 자동기록
FIFO 강제 출고(force=true) 시 anomalies 테이블에 자동 기록. 담당자·사유·Lot정보 포함.
경고 대시보드
안전재고 부족·Canister 90% 초과·Task 지연을 서버에서 집계 → 종합현황 상단 배너로 표시.
퀵메뉴
원재료 입고·출고, 부재료 입고·출고, Canister 수불을 Dashboard에서 모달로 바로 시작.
Task 관리
tasks 테이블에 title·assignee·priority·dueDate·status 저장. 상태: pending → in-progress → done → approved.
트렌드 분석
transactions를 월/주 단위로 집계. 품목별 입출고 추이를 Bar 차트로 시각화.

기능 강화 · 인터락 (안전장치)

실수·사고·무단 접근을 막는 서버 레벨 안전장치

FIFO 인터락
findEarlierLot(): 현재 Lot보다 receivedDate가 이전이고 잔량 > 0인 Lot 존재 시 409 차단. force=true만 허용.
경고 확인(ack)
경고 항목마다 확인 인원 수 카운트. 미확인 경고는 배너에 숫자로 강조하여 누락 없이 처리 유도.
Lot 1개 예외
Lot이 1개뿐이면 FIFO 검사를 건너뜀. 선택지가 없는 상황에서 불필요한 경고 방지.
입고일 필수
입고일 누락 시 FIFO receivedDate 비교 불가 → 서버 badRequest로 차단.

버그 수정 · 원인 분석

발생 원인과 해결 방법

빈 Lot 숨김
잔량 0인 Lot이 목록에 남아 FIFO 검사에서 오탐 → quantity > 0 필터 적용. 이력 조회는 별도 포함.
입고일 누락 허용
초기엔 입고일 없이도 저장 → FIFO 날짜 비교 오류 발생. required 검증 추가.
Task 마감 미처리
마감일 지난 Task가 상태 변화 없이 남아있음 → GET /tasks에서 dueDate 비교 후 자동 지연 표시.

학습 · 기술 포인트

이 버전에서 익힌 핵심 기술 개념

FIFO 알고리즘
입고일 기준 정렬 후 현재 Lot보다 오래된 잔량 존재 여부 확인. 단순 비교지만 날짜 포맷 통일이 핵심.
409 상태 코드 활용
비즈니스 규칙 위반(FIFO)에 409 Conflict 사용. 프론트에서 이 코드를 명시적으로 처리해 사용자 선택지 제공.
서버 집계 vs 클라이언트 집계
경고·통계를 서버에서 미리 집계해 클라이언트 데이터 처리 부담 제거. 큰 CSV도 한 번만 읽어 다양한 집계 동시 수행.
